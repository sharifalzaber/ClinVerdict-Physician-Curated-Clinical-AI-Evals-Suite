Engineering Recommendations for Clinical Ambient AI Pipelines
1. Decouple the Evaluation Rubric from a Single Reference Baseline
The most consistent systemic failure across this evaluation suite was Reference Bias — the judge inheriting blind spots directly from a flawed human reference note. A production-grade evaluation framework must cross-validate agent output against both the human reference AND the raw transcript independently. A judge constrained exclusively to a single baseline cannot catch what that baseline missed.
2. Mandate Informant Identity Tagging in Pediatric and Proxy Encounters
Cases #7 and #9 both demonstrated that the agent, judge, and reference note all failed to establish who was delivering the clinical history. System prompts for pediatric, geriatric, or altered-consciousness encounters must explicitly instruct the agent to isolate and tag the informant identity before extracting any clinical content. Proxy misattribution in a permanent medical record is a medico-legal liability, not a formatting error.
3. Treat Numerical Clinical Values as Non-Negotiable Extraction Targets
Case #6 demonstrated that an agent can actively misrepresent the absence of data it simply failed to extract. Glucose readings, medication doses, lab values, and vital signs must be flagged as mandatory extraction fields with a zero-tolerance threshold. Any note claiming numerical data is absent must be automatically routed for human review before EHR upload.
4. Classify Patient Dissent and Informed Refusal as Protected Tokens
Case #10 demonstrated that explicit, repetitive patient refusal can be compressed into generic boilerplate by the agent. Informed refusal, allergy declarations, and treatment objections carry direct legal weight. These must be treated as protected documentation tokens — extracted verbatim, never paraphrased, never omitted.
5. Implement a Standalone Medication Temporal Classifier
Case #4 demonstrated catastrophic chronological conflation — discontinued medications propagated as active across three SOAP sections. A dedicated temporal classification layer must distinguish between historical, discontinued, and active medications before any drug name enters the permanent record. This is not a completeness issue — it is a patient safety issue.
6. Establish an Independent Dataset Quality Audit Layer
Cases #2, #3, #5, and #8 all contained human reference notes that were themselves defective — through contamination, overreach, or omission. Any training or evaluation pipeline that treats human annotations as an unaudited gold standard will systematically inherit these defects. A mandatory pre-evaluation dataset audit step, flagging annotation inconsistencies against the raw transcript, must precede any automated scoring run.
7. Flag Assessment-Level Outputs for Mandatory Human Review
Cases #3 and #7 demonstrated agents independently generating or escalating diagnostic conclusions not spoken by the physician. The Assessment block of any AI-generated SOAP note must be flagged as a high-risk output zone requiring physician sign-off before clinical use. No agent should introduce, escalate, or confirm a diagnosis beyond what the treating physician explicitly stated.
8. Implement Clinical Trajectory Preservation as a Core Extraction Requirement
Case #3 demonstrated that stripping improving clinical trends from a note creates a false static crisis profile — driving unnecessary and costly diagnostic escalation. Agents must be explicitly instructed to extract directional language ("improving," "resolving," "worsening") as clinically significant tokens, not filler.
